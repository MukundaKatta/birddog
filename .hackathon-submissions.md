### Short description (2–3 sentences)
birddog is a Python library that puts a leash on AI scraping agents. One context manager gives you a domain allowlist, per-domain token-bucket rate caps, a JSONL audit log, and optional routing through Bright Data Web Unlocker or SERP API. Built for research bots, price trackers, and RAG ingest jobs that would otherwise burn through a Bright Data quota in 30 seconds.

### Full description

**The problem**

LLM agents have no concept of a sane scraping cadence. They wander off-domain, hammer a single host, and burn through a Bright Data quota before the first useful result comes back.

**What birddog does**

One `with bd.session()` block wraps any agent that hits the web and gives you:

1. **Domain allowlist** — wildcard subdomains (`*.example.com`), deny everything else
2. **Per-domain rate caps** — token bucket, one bucket per hostname, configurable QPS + burst
3. **JSONL audit log** — one line per fetch: url, status, bytes, elapsed_ms, session_id
4. **Bright Data Web Unlocker** — opt-in proxy that routes every fetch through Bright Data's residential network
5. **Bright Data SERP API** — the demo shows a full search→fetch pipeline: SERP results extracted via Bright Data, then depth-fetched through Web Unlocker
6. **Nimble proxy** — alternative opt-in proxy routing through ip.nimbleway.com:7000
7. **Streamlit dashboard** — live view of per-host bytes, fetch count, denials, and p50 latency from the JSONL

**How it was built**

- Core: `httpx.Client` wrapped in a context manager; token-bucket rate limiter; `AuditEvent` dataclass serialized to JSONL
- Bright Data: HTTP proxy via `brd.superproxy.io:33335` (Web Unlocker) or the SERP API endpoint for search
- Nimble: HTTP proxy via `ip.nimbleway.com:7000`
- Dashboard: `streamlit` + `pandas` reading the JSONL file
- Tests: `respx` mock transport, 15 tests covering allowlist, rate cap, Nimble proxy, and attestation
- Deployed demo: Cloud Run + Streamlit at https://birddog-demo-1030412920700.us-central1.run.app

**Bright Data integration**

The SERP + Unlock pipeline (in `examples/scrape_demo.py`):
1. Hit the Bright Data SERP API to get organic results for a query
2. For each result URL, fetch depth with Web Unlocker proxy
3. Every step logged in the JSONL with `via_brightdata: true`

The demo fixture shows 3 queries × 4 result URLs = 12 fetches, plus domain denials and rate-limit events.

### Full description (Devpost markdown)

## Inspiration

AI agents scrape the web without limits. They hammer a single host, wander off-domain, and rack up huge proxy bills before returning anything useful. I wanted a simple way to add guardrails to any agent that touches the live web.

## What it does

birddog is a Python context manager that wraps any web-scraping agent and gives it:

- **Domain allowlist** — only hosts on the list can be fetched; everything else raises `DomainDeniedError` and logs the attempt
- **Per-domain rate cap** — token bucket per hostname; configurable sustained QPS + burst
- **JSONL audit log** — one event per fetch (`fetch_ok`, `fetch_failed`, `domain_denied`, `rate_limited`, `session_open`, `session_close`)
- **Nimble proxy** — one config flag routes all fetches through `ip.nimbleway.com:7000` using your Nimble credentials
- **Streamlit dashboard** — reads the JSONL and shows per-host bytes, fetches, denials, and p50 latency

```python
from birddog import Birddog

bd = Birddog(
    allowed_domains={"*.example.com"},
    per_domain_qps=2.0,
    audit_path="runs/scrape.jsonl",
    nimble={
        "username": "account-myaccount-pipeline-mypipeline",
        "password": "...",
    },
)

with bd.session("research-bot") as s:
    r = s.fetch("https://example.com/page")
    print(r.via_nimble, r.status, r.bytes_len)
```

Every fetch routes through Nimble. The domain allowlist still applies. The audit log records `via_nimble: true` for every event.

## How I built it

- `httpx.Client` with `proxy=f"http://{username}:{password}@ip.nimbleway.com:7000"` for Nimble routing
- Token bucket class (`_Bucket`) with monotonic clock for rate limiting
- `AuditEvent` dataclass serialized to JSONL with `json.dumps(asdict(event))`
- Streamlit dashboard reads the JSONL with pandas and renders per-host tables
- 15 offline tests with `respx` mock transport — no real network required

## Challenges

Keeping `nimble` and `bright_data` mutually exclusive at the config layer without overcomplicating the dataclass. Settled on simple `elif` branching in `__enter__` — clean and explicit.

## What I learned

Nimble's HTTP proxy interface is identical to Bright Data's — same username/password auth, same HTTPS tunnel. Swapping proxy backends is genuinely a one-line config change, which validated the abstraction.

## What's next

- Async support (`httpx.AsyncClient`)
- Per-domain allowlist + rate-cap overrides
- Prometheus metrics export from the JSONL
